Operations

Recovery, validation, and runbook patterns for the platform blueprint.

Recovery

  • Data Protection Backup Vault foundation for AKS backup workflows.
  • SQL backup posture remains Azure-managed and policy-driven.
  • Restore testing must be documented before claiming production readiness.

Validation

  • Terraform formatting and validation.
  • Kustomize and Helm render checks.
  • HPA, PDB, node placement, and NAP NodePool checks.
  • SDK TypeScript compile and package dry-run checks.

Lab Checklist

  • Terraform creates the SQL-focused baseline, ACR, and GitHub Actions identity.
  • GitHub variables are bootstrapped for OIDC-based CI/CD.
  • Helm chart renders all boutique services with placement and autoscaling values.
  • Flux reconciles tenant baselines, node provisioning policy, gateway, and app releases.
  • Azure DNS and Traffic Manager records are validated before exposing application hostnames.
