HA & Autoscaling

Three capacity layers for stable platform workloads and low-cost burst traffic.

The blueprint uses on-demand capacity for the system and critical app baseline, then lets AKS Node Auto Provisioning add spot nodes for workloads that can tolerate interruption.

Capacity Model

  • System node pool: on-demand, Terraform-managed, system workloads only.
  • Baseline user node pool: on-demand, Terraform-managed, critical tenant services.
  • Spot NodePool: NAP-managed capacity for burst and non-critical services.

Workload Split

  • Baseline: frontend, cart, checkout, payment, catalog, currency, shipping.
  • Spot: ad, email, recommendation, shopping assistant, load generator.
  • Spot workloads must opt in with node selectors and tolerations.

Azure DNS

  • Public DNS zone owns portfolio and application hostnames.
  • Application records can point to Traffic Manager by CNAME.
  • Private DNS remains separate for data services such as Azure SQL private endpoints.

Traffic Manager

  • Traffic Manager provides DNS-based routing to AKS ingress endpoints.
  • One endpoint is a routing abstraction, not full regional HA.
  • Two or more independent cluster endpoints are required for real failover.

Edge Routing Flow

boutique.example.com
  -> Azure DNS CNAME
  -> Traffic Manager profile
  -> AKS Gateway public endpoint
  -> HTTPRoute
  -> boutique frontend service

Scheduling Contract

nodeSelector:
  aks-platform.azure.com/node-purpose: user
  aks-platform.azure.com/capacity: spot
tolerations:
  - key: aks-platform.azure.com/capacity
    operator: Equal
    value: spot
    effect: NoSchedule

Pod Scaling

  • HPA scales replicas from CPU or memory signals.
  • Prod critical services start with at least two replicas.
  • PDBs preserve availability during voluntary disruption.

Node Scaling

  • NAP provisions spot nodes when pending pods require capacity.
  • Classic cluster autoscaler is not used for the NAP spot layer.
  • Spot interruption is expected and limited to opted-in workloads.

Validation

kubectl kustomize k8s/platform/node-provisioning
kubectl get nodepool
kubectl get nodes -L aks-platform.azure.com/capacity
