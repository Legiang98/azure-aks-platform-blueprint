Architecture

Multi-tenant AKS platform with isolated infrastructure boundaries.

The blueprint separates Azure infrastructure, Kubernetes platform controls, app releases, database security, and platform SDK concerns.

Network

  • VPN VNet for WireGuard Portal and operator entry.
  • AKS VNet for system, app, ingress, and API server subnets.
  • Data VNet for private endpoints and database administration paths.

Kubernetes

  • Platform-owned namespaces, quotas, limit ranges, and network policies.
  • App-owned Helm chart and Flux HelmRelease manifests.
  • Shared Gateway API with namespace label-based route attachment.
  • On-demand baseline capacity with NAP-managed spot scale-out.

Capacity Layers

System workloads stay on an on-demand system node pool. Critical tenant services run on a baseline on-demand user pool, while selected burst workloads opt in to spot capacity through AKS Node Auto Provisioning.

Public Edge

Azure DNS owns the public hostname, Traffic Manager provides DNS-based routing, and Gateway API routes application traffic inside the cluster. Multi-region failover requires more than one independent endpoint.
