# Architecture Documentation Skill

## Purpose

Create or update architecture and case-study documentation for this public AKS portfolio blueprint.

## Required Structure

- Problem
- Goals
- Non-goals
- Architecture
- Components
- Security considerations
- Observability
- Deployment flow
- Rollback/recovery
- Trade-offs
- Future improvements

## Rules

- Be concise and practical.
- Explain trade-offs, not just tools.
- Avoid overclaiming production readiness.
- Use wording such as "blueprint", "reference pattern", or "demo pattern".
- Avoid client, company, tenant, subscription, private domain, or production-specific details.
- Use placeholders where implementation details are not ready.
- Keep diagrams and docs aligned when architecture changes.
